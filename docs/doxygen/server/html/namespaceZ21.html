enum class Z21::BroadcastFlags : uint32_t
strong
Enumerator
PowerLocoTurnoutChanges 

Broadcasts and info messages concerning driving and switching are delivered to the registered clients automatically. The following messages are concerned: 2.7 LAN_X_BC_TRACK_POWER_OFF 2.8 LAN_X_BC_TRACK_POWER_ON 2.9 LAN_X_BC_PROGRAMMING_MODE 2.10 LAN_X_BC_TRACK_SHORT_CIRCUIT 2.14 LAN_X_BC_STOPPED 4.4 LAN_X_LOCO_INFO (loco address must be subscribed too) 5.3 LAN_X_TURNOUT_INFO

RBusChanges 

Changes of the feedback devices on the R-Bus are sent automatically. Z21 Broadcast messages see 7.1 LAN_RMBUS_DATACHANGED

RailCOMChanges 

Changes of RailCom data of subscribed locomotives are sent automatically. Z21 Broadcast messages see 8.1 LAN_RAILCOM_DATACHANGED

SystemStatusChanges 

Changes of the Z21 system status are sent automatically. Z21 Broadcast messages see 2.18 LAN_SYSTEMSTATE_DATACHANGED

AllLocoChanges 

Extends flag 0x00000001; client now gets LAN_X_LOCO_INFO LAN_X_LOCO_INFO without having to subscribe to the corresponding locomotive addresses, i.e. for all controlled locomotives! Due to the high network traffic, this flag may only be used by adequate PC railroad automation software and is NOT intended for mobile hand controllers under any circumstances. From FW V1.20 bis V1.23: LAN_X_LOCO_INFO is sent for all locomotives. From FW V1.24: LAN_X_LOCO_INFO is sent for all modified locomotives.

LocoNetWithoutLocoAndSwitches 

Forwarding messages from LocoNet bus to LAN client without locos and switches.

LocoNetLoco 

Forwarding locomotive-specific LocoNet messages to LAN Client: OPC_LOCO_SPD, OPC_LOCO_DIRF, OPC_LOCO_SND, OPC_LOCO_F912, OPC_EXP_CMD

LocoNetSwitch 

Forwarding switch-specific LocoNet messages to LAN client: OPC_SW_REQ, OPC_SW_REP, OPC_SW_ACK, OPC_SW_STATE

LocoNetDetector 

Sending status changes of LocoNet track occupancy detectors to the LAN client. See 9.5 LAN_LOCONET_DETECTOR

RailComDataChanged 

Version 1.29: Sending changes of RailCom data to the LAN Client. Client gets LAN_RAILCOM_DATACHANGED without having to subscribe to the corresponding locomotive addresses, i.e. for all controlled locomotives! Due to the high network traffic, this flag may only be used by adequate PC railroad automation software and is NOT intended for mobile hand controllers under any circumstances. Z21 Broadcast messages see 8.1 LAN_RAILCOM_DATACHANGED

CANDetector 

Sending status changes of CAN-Bus track occupancy detectors to the LAN client. See 10.1 LAN_CAN_DETECTOR
